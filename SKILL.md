# IR文献综述写作 Skill

## 核心写作哲学

**文献不是单位，问题和张力才是单位。**

综述的任务不是"报告别人说了什么"，而是"用已有研究呈现一个结构性张力，并在这个张力中确立本研究的位置"。好的IR文献综述读起来像一个论证，而不是一份目录。

在动笔之前，先问自己：
- 这个研究试图解决的核心张力是什么？（理论困境？经验争论？概念模糊？方法缺口？）
- 已有文献在这个张力上处于什么位置？
- 综述结束时，读者应该感到"确实需要这项新研究"——是因为什么？

结构、段落组织、引用方式，都服从于对上述问题的回答。

---

## 接到任务时的判断流程

用户会提供：论文框架 + 大致文献检索方向。

**第零步：结构化提取输入信息**

在做任何张力判断之前，先从用户框架中内部确认以下五项。如有缺失，直接追问用户，不要假设：

| 提取项 | 说明 | 缺失时的追问方式 |
|---|---|---|
| 研究问题（DV） | 论文试图解释或描述什么？因变量是什么？ | "你的论文主要在解释什么现象？" |
| 核心解释与机制（IV + 机制） | 你提出的解释变量是什么？通过什么机制影响DV？ | "你认为什么因素在起作用？作用路径大致是什么？" |
| 方法与案例范围 | 量化/质性/混合？研究的时间和地区边界是什么？ | "你打算用什么方法，覆盖哪些案例或样本？" |
| 已知竞争性文献（1-3篇） | 用户自己认为最直接的竞争对手是哪几篇？ | "你知道有哪些研究已经试图回答类似问题？" |
| 核心论点（一句话） | 这篇论文想说什么？哪怕是初稿表述也行 | "用一句话说，你的论文的核心主张是什么？" |

五项全部确认后，再进入下一步。

---

**第零点五步：文献检索与核实**

提取完输入信息后，在分析张力之前，先建立真实的文献基础。综述必须建立在实际检索到的文献上，而不是训练知识的模糊印象。

**检索工具选择**

| 文献类型 | 使用工具 | 说明 |
|---|---|---|
| 英文IR文献（主流） | `gs-researcher` 或 `/gs-search` | 覆盖 APSR、IO、ISQ、JCR、WP 等顶刊 |
| 中文IR文献 | `cnki-researcher` 或 `/cnki-search` | 覆盖《世界经济与政治》《国际政治科学》等核心期刊 |
| 中英文均需要 | 两个工具分别检索 | 先英文后中文，或根据研究语境决定主次 |

**检索执行要求**

1. **根据用户提供的检索方向**，拆解为若干具体的搜索关键词组合，至少执行 2-3 轮检索（核心概念、竞争性解释、相关子领域）
2. **对用户已知的竞争性文献**，用 `/gs-search` 或 `/cnki-search` 验证其存在，用 `/gs-cited-by` 追踪引用链，确认这些文献在领域内的实际影响力
3. **检索结果处理**：从结果中筛选出直接竞争性文献（已尝试回答同一问题的）、概念性文献（界定核心概念的）、方法性文献（支撑研究设计的），丢弃弱相关条目
4. **对重要文献**，用 `/gs-fulltext` 或 `/cnki-paper-detail` 获取摘要或全文，基于实际内容而非书名判断其论点
5. 需要导出到 Zotero 时，用 `/gs-export` 或 `/cnki-export`

**文献可信度规则**

- 实际检索到并读取了摘要/全文的文献：直接使用，标注作者-年份
- 用户提供但未检索核实的文献：使用前先搜索确认，无法确认时告知用户
- 仅凭训练知识掌握的文献：若把握程度不足，用 `[待核实]` 标注具体论点，不凭印象捏造引用

检索完成后，用一个简短的文献清单告知用户找到了什么，然后进入张力分析。

**引用与文献清单预置（必须执行）**

在输出骨架前，先建立一个“候选文献表”（可在内部草稿中维护，不必全部展示给用户），每条至少包含：
- 作者（姓氏）
- 年份
- 标题
- 来源（期刊/出版社/会议）
- DOI/URL（如有）
- 使用状态：`已核实可引用` / `待核实`

后续写作时，**仅使用候选文献表中的条目进行正式引用**；凡未核实条目，一律不得作为确定性证据写入正文。

---

**第一步：识别研究类型与核心张力**

不要急着组织文献，先判断：

| 研究特征 | 核心张力通常在于 |
|---|---|
| 提出新解释变量 | 既有文献忽视了X对Y的影响 |
| 修正既有理论 | 既有解释在机制/条件/范围上存在缺陷 |
| 新概念/重新界定概念 | 既有概念模糊或错误划分了现象边界 |
| 新案例/新数据 | 既有结论的外部有效性存疑 |
| 跨层次/跨子领域 | 两个研究传统之间存在对话缺口 |

**第二步：选择写作策略**

根据张力性质，有两种基本策略，可以混用：

- **框架整合策略**：先建立一套分析坐标，再把文献放进去。适合需要重新组织既有研究、消解表面分歧、建立新分类的综述。不是"总结文献"，而是"用新坐标重新编码文献"。

- **问题拆解策略**：把一个大问题拆成若干子问题，每个子问题下并列竞争性解释，保留而非消解张力。适合争论本身就是综述重心的情况。

**第三步：确定切入方式**

开头不要从定义开始，不要从"学界对X问题已有大量研究"开始。考虑：

- **理论困境切入**：指出主流模型的有效性，然后揭示其盲区，本研究作为"修复方案"出现
- **研究演化切入**：通过研究议程的时间序列，自然引出当前焦点和本研究的位置
- **经验悖论切入**：从一个反直觉的现象或争论出发，引出理论解释的不足

**第四步：输出论证骨架，等用户确认后再动笔**

在正式写作之前，先输出一个结构骨架，格式如下，然后**等待用户确认**：

```
【文献综述骨架草案】

切入方式：[理论困境 / 研究演化 / 经验悖论]
写作策略：[框架整合 / 问题拆解 / 混合]

第1段：[段落核心主张，1句话]
  └─ 功能：[建立背景 / 确立主流解释的有效范围 / 引出核心张力]
  └─ 主要使用文献：[列出1-3篇]

第2段：[段落核心主张，1句话]
  └─ 功能：[...]
  └─ 主要使用文献：[...]

[根据需要继续]

缺口落地：既有研究在[具体点]上存在[具体问题]，因此[本研究的介入方式]。

▶ 请确认骨架方向，或指出需要调整的地方，确认后我将展开全文。
```

骨架确认后才进入正式写作，避免方向错误导致的整体返工。

## 正式写作输出规范（引用 + 参考文献 + docx交付）

正式成稿时默认输出以下内容（除非用户明确要求改格式）：

1. **文内引用（作者-年份）**
   - 中文正文示例：`……这一机制在威权信息环境下更明显（Rosenfeld & Wallace, 2024）。`
   - 英文正文示例：`…under information autocracy (Rosenfeld & Wallace, 2024).`
   - 同一论点多文献支持时：按相关性排序，通常不超过3条：`(Huang, 2018; Pan et al., 2022; Blum, 2023)`
   - 直接引用或高度依赖具体论断时，尽量给出页码：`(Wedeen, 1999, p. xx)`

2. **文末参考文献（References/参考文献）**
   - 必须提供完整列表，且**仅包含正文实际引用过的文献**。
   - 基本字段尽量齐全：作者、年份、题目、来源、卷期页码、DOI/URL（如有）。
   - 未核实完整信息时，条目后加`[待补全]`，并在文末单独提示用户补全项。

3. **引用真实性约束**
   - 严禁捏造作者、年份、题目、期刊、卷期页码、DOI。
   - 若用户要求引用某文献但当前无法核实，正文用`[待核实]`明确标记，不伪造细节。

4. **密度控制**
   - 引用服务于论证，不为堆砌而堆砌；每段通常2-4条高功能引用优先于高密度罗列。

5. **默认交付格式：docx（用户未另行指定时）**
   - 最终交付必须是 `.docx` 文件（不是仅贴纯文本）。可同时附上可复制预览文本。
   - 页面与段落格式默认值：
     - 中文字体：宋体
     - 西文字体与数字：Times New Roman
     - 字号：小四（12pt）
     - 首行缩进：2字符
     - 行间距：1.5倍
   - 推荐文件名：`IR_literature_review_YYYYMMDD.docx`（或按用户指定命名）。
   - 若用户提出学校/期刊模板，以用户模板优先覆盖上述默认值。

6. **docx生成执行要点（必须落实）**
   - 成稿后用可执行方式直接生成 `.docx`（优先使用 `python-docx`）。
   - 需同时设置西文字体与东亚字体：
     - 西文/数字：Times New Roman
     - 东亚文字：宋体（通过 eastAsia 字体位设置）
   - 段落统一设置：12pt、首行缩进约24pt（对应小四字号下2字符）、1.5倍行距。
   - 文末保留“参考文献/References”标题与条目，且与正文引用一一对应。

## 写作中的品质标准

这些是从顶刊综述中提炼的具体标准，写作时持续对照。质量控制分三个节点嵌入写作流程：

---

**节点A：骨架确认后 → 检查"缺口是否有着落"**

在用户确认骨架、准备动笔前，对每一个计划提出的缺口，自查：
- 这个缺口**从哪篇或哪类具体研究的局限中生长出来**？（来源）
- 如果不填补这个缺口，**会有什么后果**？（后果）

如果来源或后果说不清楚，缺口就是漂浮的，需要重新锚定或放弃。

---

**节点B：每段完成后 → 检查"是否出现机械列举"**

每段写完，对照以下标准检查一遍：
- 是否出现了"A认为…，B认为…，C认为…"的逐篇总结结构？如有，重组为对话形式
- 引用数量是否超过5个却没有明确论证功能？如有，合并或删除功能不明确的引用
- 段落结尾是否揭示了一个张力或缺口，供下一段接住？如没有，补充过渡逻辑

---

**节点C：全文完成后 → 检查"综述结尾是否正好是本研究开始的地方"**

全文完成后，做最终检查：
- 综述最后揭示的缺口，是否就是论文理论/实证部分要填补的那个缺口？
- 读完综述，读者能不能感受到"确实需要这项新研究"？如果感受不到，缺口表述需要加强
- 综述中有没有批评了既有文献某个问题，但论文本身并不解决这个问题的"多余攻击"？如有，删除

---

**1. 避免"机械列举"**
不要逐篇总结：「A认为…，B认为…，C认为…」
要把文献组织成对话：「关于X，一派认为…，另一派从…角度提出了不同解释，但两者都忽视了…」

**2. 缺口要有"着落"**
不要泛泛说"现有研究存在不足"。每一个缺口都要绑定具体的研究进展，让读者感到缺口是从文献内部生长出来的，而不是外加的。
- 弱写法：「现有研究对这一问题关注不足。」
- 强写法：「尽管X和Y的研究已经确立了Z机制，但当条件A不成立时，该机制是否仍然适用，尚无系统性检验。」

**3. 理论贡献要可见**
综述不只是交代背景，它本身要能体现作者的判断力。对文献的批评要有力（sharp）、有据（grounded）、有针对性（targeted）——不是"这个领域还需要更多研究"，而是"既有研究在具体的X点上存在具体的问题，本研究通过Y来解决"。

**4. 过渡要有逻辑而非形式**
段落之间的衔接不靠"此外"、"另外"、"综上所述"。靠的是：上一段结尾揭示了一个张力或缺口，下一段开头正面接住这个张力。

**5. 文献是论证的材料，不是论证本身**
引用密度不等于质量。一段综述里出现十五个引用，不如三个引用各自承担明确的论证功能。

## 子领域识别与注意事项

读取用户提供的框架后，识别子领域，注意以下写作特点：

**安全研究（威慑、冲突、联盟、军备）**
理论争论脉络清晰，流派边界分明。综述要把握"这场争论的真正分歧在哪里"——往往不是表面上的结论不同，而是假设不同或因果机制不同。

**国际政治经济学（IPE）**
实证研究密度高，方法论分歧显著（量化vs质性，国内政治vs系统层次）。综述要注意"结论的可比性"——不同研究测量的是同一件事吗？

**外交政策分析（FPA）**
跨层次分析是核心挑战。综述需要清晰标出每项研究的分析层次（个人/官僚/国内/系统），并指出层次间的整合缺口。

**国际制度/国际组织**
大量委托-代理框架，近年微观基础转向明显。注意区分"组织作为行为体"和"个体官僚作为行为体"两类研究的不同假设。

**比较威权/信息政治**
跨学科性强（IR+比较政治）。概念界定是高频争议点（如"宣传"与"审查"的边界）。综述要对概念争议有所交代，而不是直接跳进实证研究。

**建构主义/规范研究**
本体论和认识论立场往往直接影响研究设计，不能假装这些分歧不存在。综述要能准确还原不同立场的内在逻辑，而不只是描述结论。
