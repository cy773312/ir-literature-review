# IR文献综述写作 Skill

## 核心写作哲学

**文献不是单位，问题和张力才是单位。**

综述的任务不是"报告别人说了什么"，而是"用已有研究呈现一个结构性张力，并在这个张力中确立本研究的位置"。好的IR文献综述读起来像一个论证，而不是一份目录。

在动笔之前，先问自己：
- 这个研究试图解决的核心张力是什么？（理论困境？经验争论？概念模糊？方法缺口？）
- 已有文献在这个张力上处于什么位置？
- 综述结束时，读者应该感到"确实需要这项新研究"——是因为什么？

结构、段落组织、引用方式，都服从于对上述问题的回答。

---

## 接到任务时的判断流程

用户最少应提供：**文献综述检索范围**。可选提供：论文框架、初步文献（种子文献）、格式要求。

**第零步：结构化提取输入信息（先检索，后定位）**

在做任何张力判断之前，先确认以下六项。如有缺失，直接追问用户，不要假设：

| 提取项 | 说明 | 缺失时的追问方式 |
|---|---|---|
| 检索主题与边界 | 研究对象、核心概念、时间/地区/案例边界 | "这次综述聚焦什么问题？时间和地区边界是什么？" |
| 语种与数据库优先级 | 中文/英文比例，CNKI 与 GS 主次 | "你希望中文文献、英文文献各占多少？先检索哪一边？" |
| 纳入与排除标准 | 顶刊/核心期刊、年份窗口、是否排除评论性文章 | "你希望优先哪些期刊与年份？有哪些文献类型要排除？" |
| 初步文献（种子） | 用户手头已有的 2-10 篇相关文献 | "你手里已有哪几篇文献？请给标题或作者-年份。" |
| 论文框架（可选） | 若已成型可提供，用于后续研究定位 | "如果你已有论文框架，可以发我；没有也可以先做检索综述。" |
| 输出目标 | 仅检索底稿 / 综述草稿 / 直接成稿 docx | "这轮你要检索底稿，还是直接写成综述正文？" |
| 质量档位 | `standard`（默认）/ `rigorous`（更深检索）/ `fast`（快速草稿） | "你希望标准质量、深度质量，还是快速草稿？" |

确认后先进入检索与核实，**不要先输出综述骨架**。默认质量档位为 `standard`。

---

**第零点五步：文献检索与核实（强制）**

提取完输入信息后，在分析张力之前先建立真实文献基础。综述必须建立在已检索并核实的文献上，而不是训练知识的模糊印象。

**检索工具选择**

| 文献类型 | 使用工具 | 说明 |
|---|---|---|
| 英文IR文献（主流） | `gs-researcher` 或 `/gs-search` | 覆盖 APSR、IO、ISQ、JCR、WP 等顶刊 |
| 中文IR文献 | `cnki-researcher` 或 `/cnki-search` | 覆盖《世界经济与政治》《国际政治科学》等核心期刊 |
| 中英文均需要 | 两个工具分别检索 | 按用户指定语种比例与优先级执行 |

实际检索工作会优先依赖这两个外部 skill：

- [gs-skills](https://github.com/cookjohn/gs-skills)
- [cnki-skills](https://github.com/cookjohn/cnki-skills)

**检索执行要求（跨用户/跨模型一致性约束）**

1. 先拆解关键词矩阵：核心概念、同义词、竞争机制、子议题，各组关键词都必须覆盖
2. 按质量档位执行最小检索配额：
   - `fast`：每个数据库 ≥ 2 轮检索，候选文献 ≥ 12
   - `standard`：每个数据库 ≥ 3 轮检索，候选文献 ≥ 20
   - `rigorous`：每个数据库 ≥ 4 轮检索，候选文献 ≥ 30
3. 对用户提供的初步文献逐条核实，用 `/gs-search` 或 `/cnki-search` 确认条目，并用 `/gs-cited-by` 或 CNKI 引文链扩展高相关文献
4. 检索结果必须按功能分类：直接竞争性、概念界定、方法支撑、背景补充；每类至少保留 2 篇（若确实不足，要显式说明）
5. 对核心文献用 `/gs-fulltext` 或 `/cnki-paper-detail` 读取摘要/全文后再判断贡献，不得仅凭标题
6. 需要导出 Zotero 时，用 `/gs-export` 或 `/cnki-export`

**文献可信度规则**

- 已检索并读取摘要/全文的文献：可直接使用，标注作者-年份
- 用户提供但未检索核实的文献：使用前先确认，无法确认时明确告知用户
- 仅凭训练知识掌握的文献：用 `[待核实]` 标注，不凭印象捏造引用

**引用与文献清单预置（必须执行）**

在任何写作输出前，先建立一个“候选文献表”（可在内部草稿中维护，不必全部展示给用户），每条至少包含：
- 作者（姓氏）
- 年份
- 标题
- 来源（期刊/出版社/会议）
- DOI/URL（如有）
- 使用状态：`已核实可引用` / `待核实`

后续写作时，**仅使用候选文献表中的“已核实可引用”条目进行正式引用**；未核实条目不得作为确定性证据写入正文。

**一致性输出约束（必须执行）**

为降低不同模型、不同 agent、不同用户输入风格造成的波动，检索阶段一律按以下固定模板输出：

1) 任务边界（主题/时间/地区/语种配比）  
2) 检索日志（数据库、关键词组、轮次）  
3) 种子文献核实结果（已核实/未核实）  
4) 文献簇与争议点（按议题聚类）  
5) 初步缺口（最多 2-3 条）  
6) 下一步动作（写作或补检索）

禁止省略 2)、3)、4) 这三项。

---

**第一步：先输出“文献检索范围与文献地图”（先于骨架）**

检索完成后，先输出结构化检索结果并等待用户确认：

```
【文献检索范围与文献地图】

检索边界：
- 主题与问题：
- 时间窗口：
- 地区/案例范围：
- 数据库与语种配比：
- 质量档位：`fast / standard / rigorous`

检索日志（可复现）：
- GS：第1轮关键词[...]；第2轮关键词[...]；第3轮关键词[...]
- CNKI：第1轮关键词[...]；第2轮关键词[...]；第3轮关键词[...]

初步文献处理结果：
- 已核实：
- 未核实（需补充信息）：

核心文献簇（按议题）：
1) [议题A]：代表文献[...]；主要结论[...]；争议点[...]
2) [议题B]：代表文献[...]；主要结论[...]；争议点[...]

初步研究缺口（基于已检索文献）：
- 缺口1：
- 缺口2：

稳定性自检：
- 是否满足最小检索配额：是/否
- 是否覆盖竞争性/概念/方法三类核心文献：是/否
- 是否存在未核实引用进入正文风险：有/无

▶ 请先确认检索范围与文献清单是否需要增删。确认后我再进行研究定位与综述写作。
```

---

**第二步：识别研究类型与核心张力（基于已确认文献）**

不要急着组织文献，先判断：

| 研究特征 | 核心张力通常在于 |
|---|---|
| 提出新解释变量 | 既有文献忽视了X对Y的影响 |
| 修正既有理论 | 既有解释在机制/条件/范围上存在缺陷 |
| 新概念/重新界定概念 | 既有概念模糊或错误划分了现象边界 |
| 新案例/新数据 | 既有结论的外部有效性存疑 |
| 跨层次/跨子领域 | 两个研究传统之间存在对话缺口 |

**第三步：选择写作策略与切入方式**

根据张力性质，有两种基本策略，可以混用：

- **框架整合策略**：先建立一套分析坐标，再把文献放进去。适合需要重新组织既有研究、消解表面分歧、建立新分类的综述。不是"总结文献"，而是"用新坐标重新编码文献"。

- **问题拆解策略**：把一个大问题拆成若干子问题，每个子问题下并列竞争性解释，保留而非消解张力。适合争论本身就是综述重心的情况。

开头不要从定义开始，不要从"学界对X问题已有大量研究"开始。考虑：

- **理论困境切入**：指出主流模型的有效性，然后揭示其盲区，本研究作为"修复方案"出现
- **研究演化切入**：通过研究议程的时间序列，自然引出当前焦点和本研究的位置
- **经验悖论切入**：从一个反直觉的现象或争论出发，引出理论解释的不足

**第四步：按需输出骨架后再动笔（骨架不再前置）**

默认流程下，确认“文献检索范围与文献地图”后即可进入写作；只有在用户明确要求“先看骨架”时，才输出骨架并等待确认。

**第五步：质量闸门与回退机制（稳健性关键）**

在正文输出前，必须先过以下闸门：

- 闸门A（证据闸门）：正文拟使用文献中，`已核实可引用` 占比应为 100%
- 闸门B（结构闸门）：每段必须有“主张 + 证据 + 过渡”三要素，避免堆引文
- 闸门C（定位闸门）：结尾缺口必须可直接映射到用户研究问题

若任一闸门未通过，执行回退：
1) 先补检索（优先补竞争性或方法性文献）
2) 更新文献地图并重新确认
3) 再进入正文

**第六步：agent / 模型无关执行规则（保证可迁移质量）**

为保证不同用户、不同 agent、不同模型下结果接近，执行层可替换，但产出协议不可变：

- 一致性机制的目标是**抬高质量下限**，不是压低质量上限；协议只规定最低要求，不限制更强模型做更深入的论证与综合
- 检索执行层可用：`gs-researcher`、`cnki-researcher`、`/gs-search`、`/cnki-search`（按可用性选择）
- 无论使用哪个执行层，都必须回填同一“检索日志 + 文献地图 + 稳定性自检”模板
- 不允许使用“模型主观判断”替代检索证据；任何关键论断都要可追溯到已核实文献
- 输出质量按同一 Rubric 自检（每项 0-2 分，总分 10）：
  1) 检索覆盖度（关键词矩阵与轮次）
  2) 文献核实度（种子与核心文献核实）
  3) 争议提炼度（是否形成可辩论张力）
  4) 研究定位清晰度（缺口可映射论文问题）
  5) 引用合规度（无未核实硬引用）
- `standard` 档位最低通过分：8/10；`rigorous`：9/10；未达标则回退补检索
- 达到最低分后，若模型能力允许，应继续优化“争议提炼度”“研究定位清晰度”和段落论证力度，不做上限封顶

## 正式写作输出规范（引用 + 参考文献 + docx交付）

正式成稿时默认输出以下内容（除非用户明确要求改格式）：

1. **文内引用（作者-年份）**
   - 中文正文示例：`……这一机制在威权信息环境下更明显（Rosenfeld & Wallace, 2024）。`
   - 英文正文示例：`…under information autocracy (Rosenfeld & Wallace, 2024).`
   - 同一论点多文献支持时：按相关性排序，通常不超过3条：`(Huang, 2018; Pan et al., 2022; Blum, 2023)`
   - 直接引用或高度依赖具体论断时，尽量给出页码：`(Wedeen, 1999, p. xx)`

2. **文末参考文献（References/参考文献）**
   - 必须提供完整列表，且**仅包含正文实际引用过的文献**。
   - 基本字段尽量齐全：作者、年份、题目、来源、卷期页码、DOI/URL（如有）。
   - 未核实完整信息时，条目后加`[待补全]`，并在文末单独提示用户补全项。

3. **引用真实性约束**
   - 严禁捏造作者、年份、题目、期刊、卷期页码、DOI。
   - 若用户要求引用某文献但当前无法核实，正文用`[待核实]`明确标记，不伪造细节。

4. **密度控制**
   - 引用服务于论证，不为堆砌而堆砌；每段通常2-4条高功能引用优先于高密度罗列。

5. **默认交付格式：docx（用户未另行指定时）**
   - 最终交付必须是 `.docx` 文件（不是仅贴纯文本）。可同时附上可复制预览文本。
   - 页面与段落格式默认值：
     - 中文字体：宋体
     - 西文字体与数字：Times New Roman
     - 字号：小四（12pt）
     - 首行缩进：2字符
     - 行间距：1.5倍
   - 推荐文件名：`IR_literature_review_YYYYMMDD.docx`（或按用户指定命名）。
   - 若用户提出学校/期刊模板，以用户模板优先覆盖上述默认值。

6. **docx生成执行要点（必须落实）**
   - 成稿后用可执行方式直接生成 `.docx`（优先使用 `python-docx`）。
   - 需同时设置西文字体与东亚字体：
     - 西文/数字：Times New Roman
     - 东亚文字：宋体（通过 eastAsia 字体位设置）
   - 段落统一设置：12pt、首行缩进约24pt（对应小四字号下2字符）、1.5倍行距。
   - 文末保留“参考文献/References”标题与条目，且与正文引用一一对应。

## 写作中的品质标准

这些是从顶刊综述中提炼的具体标准，写作时持续对照。质量控制分三个节点嵌入写作流程：

---

**节点A：检索地图确认后（若用户要求，再加骨架确认）→ 检查"缺口是否有着落"**

在用户确认检索地图、准备动笔前（若有骨架则在骨架确认后），对每一个计划提出的缺口，自查：
- 这个缺口**从哪篇或哪类具体研究的局限中生长出来**？（来源）
- 如果不填补这个缺口，**会有什么后果**？（后果）

如果来源或后果说不清楚，缺口就是漂浮的，需要重新锚定或放弃。

---

**节点B：每段完成后 → 检查"是否出现机械列举"**

每段写完，对照以下标准检查一遍：
- 是否出现了"A认为…，B认为…，C认为…"的逐篇总结结构？如有，重组为对话形式
- 引用数量是否超过5个却没有明确论证功能？如有，合并或删除功能不明确的引用
- 段落结尾是否揭示了一个张力或缺口，供下一段接住？如没有，补充过渡逻辑

---

**节点C：全文完成后 → 检查"综述结尾是否正好是本研究开始的地方"**

全文完成后，做最终检查：
- 综述最后揭示的缺口，是否就是论文理论/实证部分要填补的那个缺口？
- 读完综述，读者能不能感受到"确实需要这项新研究"？如果感受不到，缺口表述需要加强
- 综述中有没有批评了既有文献某个问题，但论文本身并不解决这个问题的"多余攻击"？如有，删除

---

**1. 避免"机械列举"**
不要逐篇总结：「A认为…，B认为…，C认为…」
要把文献组织成对话：「关于X，一派认为…，另一派从…角度提出了不同解释，但两者都忽视了…」

**2. 缺口要有"着落"**
不要泛泛说"现有研究存在不足"。每一个缺口都要绑定具体的研究进展，让读者感到缺口是从文献内部生长出来的，而不是外加的。
- 弱写法：「现有研究对这一问题关注不足。」
- 强写法：「尽管X和Y的研究已经确立了Z机制，但当条件A不成立时，该机制是否仍然适用，尚无系统性检验。」

**3. 理论贡献要可见**
综述不只是交代背景，它本身要能体现作者的判断力。对文献的批评要有力（sharp）、有据（grounded）、有针对性（targeted）——不是"这个领域还需要更多研究"，而是"既有研究在具体的X点上存在具体的问题，本研究通过Y来解决"。

**4. 过渡要有逻辑而非形式**
段落之间的衔接不靠"此外"、"另外"、"综上所述"。靠的是：上一段结尾揭示了一个张力或缺口，下一段开头正面接住这个张力。

**5. 文献是论证的材料，不是论证本身**
引用密度不等于质量。一段综述里出现十五个引用，不如三个引用各自承担明确的论证功能。

---

## 子领域识别与注意事项

读取用户提供的检索范围与（如有）论文框架后，识别子领域，注意以下写作特点：

**安全研究（威慑、冲突、联盟、军备）**
理论争论脉络清晰，流派边界分明。综述要把握"这场争论的真正分歧在哪里"——往往不是表面上的结论不同，而是假设不同或因果机制不同。

**国际政治经济学（IPE）**
实证研究密度高，方法论分歧显著（量化vs质性，国内政治vs系统层次）。综述要注意"结论的可比性"——不同研究测量的是同一件事吗？

**外交政策分析（FPA）**
跨层次分析是核心挑战。综述需要清晰标出每项研究的分析层次（个人/官僚/国内/系统），并指出层次间的整合缺口。

**国际制度/国际组织**
大量委托-代理框架，近年微观基础转向明显。注意区分"组织作为行为体"和"个体官僚作为行为体"两类研究的不同假设。

**比较威权/信息政治**
跨学科性强（IR+比较政治）。概念界定是高频争议点（如"宣传"与"审查"的边界）。综述要对概念争议有所交代，而不是直接跳进实证研究。

**建构主义/规范研究**
本体论和认识论立场往往直接影响研究设计，不能假装这些分歧不存在。综述要能准确还原不同立场的内在逻辑，而不只是描述结论。
